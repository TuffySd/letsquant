# 策略验证手册

## 设计原则

当前资金规模为 10 万元人民币，策略应优先控制错误成本：

- 避免过多分散。单票 15%-20%、最多 4-5 只股票更适合当前资金量。
- 避免高换手。佣金最低收费和人工执行都会放大频繁交易的摩擦。
- 避免只看收益率。每次迭代必须同时看最大回撤、收益回撤比、交易次数、胜率和平均盈亏。
- 避免未来函数。信号只能使用当日收盘前已经存在的数据，默认下一交易日开盘执行。

## 初始策略

`TrendBreakoutStrategy` 是一个中短期趋势突破基准策略：

- 趋势过滤：短、中、长期均线多头排列。
- 突破确认：收盘价突破前一段时间高点。
- 动量确认：近 N 日收益率高于阈值。
- 流动性确认：当日成交量不低于近期均量阈值。
- 风险退出：固定止损、移动止盈、跌破中期均线、最长持仓天数。

这个策略不是最终答案，而是一个稳定基线。后续任何新策略都需要和它在同一数据、同一成本、同一股票池下做对照。

## 每日人工执行流程

1. 收盘后更新日线数据。
2. 运行 `letsquant signal` 生成 `results/current_signals.csv`。
3. 人工检查是否停牌、涨跌停、公告、业绩预告、重大事件。
4. 次日开盘后在券商软件按信号执行。
5. 收盘后记录真实成交价，后续可补充实盘复盘模块。

## 回测必须补齐的真实市场约束

后续接入正式数据源后，应逐步增加以下过滤：

- ST、退市整理、北交所等是否纳入。
- 停牌、涨跌停导致无法成交。
- 新股上市未满一定天数。
- 日均成交额过低的股票。
- 前复权/后复权价格一致性。
- 指数基准，如沪深 300、中证 500、中证 1000。

## 迭代纪律

每次策略变更都应记录：

- 变更目的。
- 变更参数。
- 样本内表现。
- 样本外表现。
- 最大回撤变化。
- 成交次数变化。
- 是否增加了不可解释的复杂度。
